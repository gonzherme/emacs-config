*Native Compilation*
=emacs-plus= comes by default with Native Compilation. If switch =emacs= version, make sure to download with native compilation

Directly translates Emacs Lisp (Elisp) code into machine code
+ massive performance boost (2x to 5x)
+ smoother text rendering
+ much faster package execution (LSP integration)
+ everything will feel snappier, reducing UI stutter
+ faster startup time
** OS Integration & Paths
#+begin_src emacs-lisp
;; macOS GUI applications do not naturally inherit environment variables (like $PATH)
;; from your terminal. This package forces Emacs to read your shell's profile
;; so tools like node, python, and go work as expected.
;; NOTE: This will add a slight delay to startup as it has to spin up an interactive shell.
(use-package exec-path-from-shell
  :config
  (exec-path-from-shell-initialize))


;; Tell Emacs where to look for Homebrew binaries and GNU utilities on macOS
(when (eq system-type 'darwin)
  ;; macOS defaults to older BSD utilities. If you installed GNU coreutils via Homebrew 
  ;; (for tools like a modern `ls` or `gls`), Emacs needs to know where they live.
  ;; We target the Apple Silicon specific Homebrew path.
  (let ((gnu-path "/opt/homebrew/opt/coreutils/libexec/gnubin"))    
    ;; Check if the directory actually exists so it doesn't throw errors on Intel Macs or Linux
    (when (file-directory-p gnu-path)      
      ;; 1. Update the internal terminal's $PATH so tools run inside Emacs find GNU utils first
      (setenv "PATH" (concat gnu-path ":" (getenv "PATH")))      
      ;; 2. Update Emacs's own command search path so it natively uses GNU utils over macOS defaults
      (add-to-list 'exec-path gnu-path))))
#+end_src

integration mac reminders
#+begin_src emacs-lisp
(define-derived-mode apple-reminders-mode org-mode "Reminders"
  "A specialized Org mode that automatically fetches Apple Reminders."
  (let* ((cli-path (or (executable-find "remindctl")
                       "/opt/homebrew/bin/remindctl"))
         (inhibit-read-only t))
    
    (unless (file-exists-p cli-path)
      (error "The remindctl CLI was not found."))
      
    ;; Fetch and insert the reminders
    (erase-buffer)
    ;; file formatting
    (insert "📅 Reminders Due Today\n")
    
    (let ((output (shell-command-to-string (concat cli-path " today"))))
      (if (string-blank-p output)
          (insert "No reminders due today! 🎉\n")
        (insert output)))
        
    (goto-char (point-min))
    
    ;; Trick Emacs into thinking the file is unmodified so it doesn't ask to save
    (set-buffer-modified-p nil))
    
  ;; Lock the file as read-only
  (read-only-mode 1))

;; Tell Emacs to strictly use this mode whenever a file named "reminders.org" is opened
(add-to-list 'auto-mode-alist '("/reminders\\.org\\'" . apple-reminders-mode))

** Package Management
- *Package manager*: the software that manages the installation and updating of packages. I am using the standard built-in emacs package manager =package=
- *Archive*: online repositories where packages are stored. I'm using two archives: =elpa= (maintained by GNU developers) and =melpa= (open source, less strict, largest emacs archive)

#+begin_src emacs-lisp
;; import package
(require 'package)

;; set package repository sources
(setq package-archives '(("melpa" . "https://melpa.org/packages/")
                         ("elpa" . "https://elpa.gnu.org/packages/")))

;; fire up the package system
(package-initialize)
(unless package-archive-contents
  (package-refresh-contents))

;; if we are on a fresh install, download use-package first
(unless (package-installed-p 'use-package)
  (package-install 'use-package))

;; import use-package
(require 'use-package)
;; automatically download packages if not present
(setq use-package-always-ensure t)
#+end_src

Cmd + K to insert link
#+begin_src emacs-lisp
;; insert a link. If text is selected, use it as the description and skip the prompt."
(with-eval-after-load 'org
  (defun my-org-insert-link-dwim ()
    (interactive)
    (if (use-region-p)
        (let ((beg (set-marker (make-marker) (region-beginning)))
              (end (set-marker (make-marker) (region-end)))
              (region-text (buffer-substring-no-properties (region-beginning) (region-end))))
          ;; Passing region-text as the 3rd argument tells Org to skip the description prompt
          (org-insert-link nil nil region-text)
          ;; Clean up the old plain text that was highlighted
          (delete-region beg end)
          (set-marker beg nil)
          (set-marker end nil))
      ;; If no text is selected, fall back to standard behavior
      (call-interactively #'org-insert-link)))

  ;; Bind your shortcut to the new wrapper function
  (define-key org-mode-map (kbd "M-k") #'my-org-insert-link-dwim)
)

** lsp
*** lsp concept
Language Server Protocol (LSP), introduced by Microsoft in 2016.Before, features like auto-complete, go-to-definition or syntax checking had to be written in plugins, different ones for different editors.

Instead of writing a unique plugin for every editor-language combination, a language community only needs to write one LSP server

*Client and Server*
- LSP Client: your text editor (VSCode, Emacs, ...)
- LSP Server: a separate background process that deeply understands the language. Developed directly by the developers of the programming language. For example: =pyright= for Python, =gopls= for Go, =tsserver= for TypeScript

*Real-time*
The text editor and LSP server processes are in constant communication. When you open a file like =main.py=, your editor talks to the LSP server process to get syntax highlighting and all the features. When you interact with the code, like hovering over a variable to see its type, your editor sends the LSP server process an event like "get type info of variable =x=", and LSP server process will analyze and answer.

*Different Language Features*
Different languages will have different LSP server implementations, so they will most likely have different features.

*** lsp-mode
[[https://emacs-lsp.github.io/lsp-mode/page/languages/][Documentation]] on each langauge LSP servers in emacs

** tree-sitter
*Concept*
- Historically, text editors relied RegEx to figure out how to highlight your code, indent it, or fold it

- [[https://www.youtube.com/watch?v=Jes3bD6P0To][talk on tree-sitter]]

*Benefits*
- better highlighting
- better indentation
- better code folding

(defun my-org-image-click-fix (&rest _args)
  "Add a keymap to Org image overlays to allow opening with a single left click."
  (let ((image-map (make-sparse-keymap)))
    ;; Bind left-click to move the cursor to the mouse and open the link

** dired-sidebar
- much faster, less clunky than treemacs
#+begin_src emacs-lisp
;; icon setup
(use-package vscode-icon
  :commands (vscode-icon-for-file))

;; dired-subtree (inline directory expansion)
(use-package dired-subtree
  :after dired
  :bind (:map dired-mode-map
              ("TAB" . dired-subtree-toggle)
              ("<tab>" . dired-subtree-toggle)
              ("<backtab>" . dired-subtree-cycle))
  :config
  ;; Customizes the indentation of nested folders for a cleaner, modern look
  (setq dired-subtree-line-prefix "  "))

;; sidebar setup
(use-package dired-sidebar
  :bind (("C-t" . dired-sidebar-toggle-sidebar))
  :commands (dired-sidebar-toggle-sidebar)
  :init
  (add-hook 'dired-sidebar-mode-hook
            (lambda ()
              (unless (file-remote-p default-directory)
                (auto-revert-mode 1))
              ;; Hide file permissions, sizes, and owners for a minimal look
              (dired-hide-details-mode 1)
              ;; Remove the mode line inside the sidebar window for a seamless panel look
              (setq-local mode-line-format nil)
              ;; Optional: Disable line numbers if you use global-display-line-numbers-mode
              (display-line-numbers-mode -1)))
  :config
  ;; Tell dired-sidebar to vscode icons
  (setq dired-sidebar-theme 'vscode)
  ;; Perfect sidebar width
  (setq dired-sidebar-width 30)
  ;; Automatically focus/reveal the current active file in the sidebar
  (setq dired-sidebar-should-follow-file t))
#+end_src

*  run last
Emacs has a notoriously low default Garbage Collection (GC) threshold (about 800KB). Because loading packages generates a lot of garbage, Emacs will pause hundreds of times during startup just to clean up memory.

Garbage collection frequency was reduced in =early-init.el= for faster startup, so need to reset now:
#+begin_src emacs-lisp
(setq gc-cons-threshold (* 16 1024 1024))
#+end_src
